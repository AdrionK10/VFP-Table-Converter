QUICK INSTALLATION:

 

1.      If you've gotten this far, then you have already executed the CEE60.EXE file and several files were extracted.  We recommend that you put these files in a subdirectory called CEE6 beneath your VFP home directory (e.g. \VFP98\CEE6).  If you choose to put CEE in another directory, be sure to include that directory in your VFP search path.

2.      From the command window, type: DO HOME()+"CEE6\CEE6". You will probably want to modify VFPSTART.PRG to load CEE automatically every time you load VFP.  Refer to the CEE documentation for instructions on doing this.

3.      The first time you run CEE, you will be prompted to Locate or Create the EXPAND.DBF file (which is used for macro expansions).  You should choose Create unless you already were using CEE5, then you can point back to that directory and use the Expand you created in CEE5.

4.      Since this is the first time CEE is run, you will be asked if you want to configure CEE.  If you answer Yes,  you are presented the CEE configuration screen.

5.      That's it!  To jump right into the CEE feature set bring up CEE menu , bring up an editing window.

 

CHANGES IN VERSION 6.0:

 

-         Recompiled to work with VFP 6.0

-         You can set the VFP comment string within our configuration screen

-         For our commenting feature, you can define a comment header that gets inserted (for example, to document the date/time the piece of code was commented out)

-         When we do macro expansion, we're now smarter about tabs vs. spaces.  If you always use tabs within your macro definition, then we can optionally convert those tabs to x number of spaces on expansion

 

SAMPLING OF NEW / PREVIOUS FEATURES:

 

-         Open the CEE Configuration page by clicking on CEE / Configure.
Note the Comment header under the CEE Comment String definition.
If you add something like “* Changes by ##INITIALS## <<TTOC(DATETIME())>>  “, you will get a customized comment header for each time you use the CEE comment tool, like this:. 
*.* Changes by RCW 12/02/1998 05:04:47 PM

-         Within the method editor, automatically insert the parent method's parameters with the Grab Parameters feature.

-         Default values for expansion prompts.  For example, you type DOW<space> and it asks for your DO WHILE expression, but defaults to ".T."  Below is an example of specifying a default value for the Message prompt in the WAW macro:

 

        WAIT WINDOW "##Message->Your message goes here##" ~

 

-         You can now have a second Expansion table.  Now, why would you want a another expand table?  For team development, so that you have a shared and a personal copy of the expand table.  This is useful in team environments, especially in enforcing commenting conventions.

-         Automatically add the variable at the current cursor position to your LOCAL or PRIVATE list (a new LOCAL/PRIVATE statement is created if one doesn't already exist for the current function/procedure/method).

-         Automatically open include files when positioned on a  #include line.

-         Change the case of selected text to upper, lower, or proper.

-         Use CTRL-TAB to cycle only thru your editing windows. Unlike Ctrl-F1, it by-passes the command window, the property sheet, and other non-editing windows.

-         Use the Compressed View feature to display only lines of text that match your search criteria.

-         The Brace Matching features makes it easy to determine if your parenthesis match up.
